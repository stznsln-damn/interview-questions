# 怎么根据当前滚动位置，实现目录中对应节点自动高亮？

## 需求拆解

用户滚动页面时，右侧目录（TOC）中当前可见章节自动高亮。核心问题：如何判断当前滚动位置对应哪个章节。

## 方案一：IntersectionObserver（推荐）

浏览器原生能力，不需要手动计算位置，不阻塞主线程。

核心思路：观察每个章节标题元素，当标题进入/离开视口时更新高亮。

关键配置 `rootMargin: '0px 0px -80% 0px'`：将检测区域限制在视口顶部 20%，只有标题滚动到该区域才算"当前章节"，符合阅读位置直觉。

从所有可见标题中，选择文档顺序最靠前的那个作为当前高亮。

## 方案二：scroll 事件 + offsetTop 计算

传统方案，兼容性最好。监听滚动事件，遍历所有标题找到最后一个 offsetTop <= scrollTop + 阈值的标题。

需用 requestAnimationFrame 节流 + passive: true 优化性能。

## 两种方案对比

| 特性        | IntersectionObserver         | scroll + offsetTop             |
| ----------- | ---------------------------- | ------------------------------ |
| 性能        | 浏览器原生优化，不阻塞主线程 | 需手动节流，频繁触发           |
| 触发 reflow | 不会                         | offsetTop 读取触发 layout 计算 |
| 精确度      | 基于可见性，天然准确         | 需手动调阈值                   |
| 兼容性      | IE 不支持（可 polyfill）     | 全兼容                         |

## 关键细节

### 1. 点击目录跳转时的冲突

点击目录触发 scrollTo 后，滚动过程中 Observer 会不断更新 activeId 导致闪烁。解决：用 ref 标记"正在点击跳转"，跳转期间跳过 Observer 回调。

### 2. 页面底部章节无法高亮

最后一个章节内容很短时标题永远无法到达检测区域。解决：检测是否滚到页面底部，如果是则直接高亮最后一个标题。

### 3. 动态内容

懒加载图片等导致标题位置变化。scroll 方案缓存的 offsetTop 会失效需重新计算，IntersectionObserver 不受此影响。

## 总结

1. **推荐 IntersectionObserver**：性能最优，不触发 reflow，代码简洁
2. **核心技巧**：通过 rootMargin 将检测区域限制在视口顶部，匹配阅读位置
3. **scroll 方案作为降级**：requestAnimationFrame 节流 + passive 优化
4. **细节处理**：点击跳转锁定防闪烁、页面底部兜底、动态内容适配
